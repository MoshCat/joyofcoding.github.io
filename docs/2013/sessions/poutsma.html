The Poutsma Principles

The name "Poutsma Principle" goes back to a SpringSource engineering meeting that was held in the summer of 2008. While watching a presentation about the API of a new Spring portfolio project that was to be introduced, I blurted out something along the lines of "I don't understand it, and that probably means it is wrong." Later on, the term "Poutsma Principle" remained in use within SpringSource to make sure that our APIs were as simple as possible.

In this talk, I plan to present this first and other Poutsma Principles: essentially things I learned while working on the Spring Framework for over seven years. It is meant to be an interactive session, with lots of discussion, disagreement and more.

Arjen Poutsma

Arjen Poutsma is a Staff Engineer at SpringSource (a division of VMware) with more than fifteen years' experience in commercial software environments. During this time he has worked with both Java EE and Microsoft .NET.

In 2004, Arjen started to specialise in Web Services and Service Oriented Architectures. During this period he has conducted trainings and has researched SOAs in large organisations.

Arjen is the founder and the project lead for the Spring Web Services. This Spring project aims at facilitating development of document-driven web services. He worked on the REST support in Spring 3.0 and 3.1. Recently, Arjen started the Spring Scala project, aimed to make using Spring in Scala easier.
